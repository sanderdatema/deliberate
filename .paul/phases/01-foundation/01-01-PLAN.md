<objective>
## Goal
Create the persona definition format (YAML with hard constraints per thinker), a starter set of 8 personas, and a basic `/deliberate` slash command that can spawn a single debater agent and return its perspective.

## Purpose
This is the foundation: without well-defined personas with hard reasoning constraints, the entire system falls flat. The personas ARE the product — they enforce cognitive diversity by forbidding specific reasoning patterns per thinker, not just suggesting styles.

## Output
- 8 persona YAML files with hard constraints
- A schema definition for persona validation
- A basic `/deliberate` slash command (single-round, parallel agents)
- Validation tests for persona format
- Project CLAUDE.md with development conventions
</objective>

<context>
## Project Context
@.paul/PROJECT.md
@.paul/ROADMAP.md

## Inspiration
Martijn Aslanders "Magische Dertien" system:
- Each figure has **hard constraints that FORBID a specific reasoning pattern** — not style suggestions but fundamental limitations
- Socrates: may NOT take a position, only ask questions
- Occam: may NOT add complexity, only remove
- Holmes: may NOT use analogies, only evidence
- Lupin: MUST invert every position as instinctive reflex
- Da Vinci: MUST find cross-domain structural patterns
- Marx (editor): identifies collective blind spots — assumptions everyone shares
- Hegel (editor): seeks synthesis that transcends oppositions (Aufhebung)
- Arendt (editor): discovers the MECHANISMS that produce blind spots

Key insight: the power is in **question-shifting** — the system produces better questions, not just better answers.
</context>

```gherkin

When all 8 YAML files are present (socrates, occam, da-vinci, holmes, lupin, marx, hegel, arendt)
Then each has a unique reasoning style with at least 2 hard constraints in forbidden

And each system_prompt enforces its constraints with "You MUST NOT" / "You are FORBIDDEN from" language
```

```gherkin

When the command executes
Then it loads all persona YAML files from personas/
And spawns one Agent per analyst persona in parallel
And collects their responses
And spawns editor personas sequentially with all analyst output as context
And presents a formatted markdown report with each perspective labeled
```

    Create the Claude Code slash command at .claude/commands/deliberate.md.

    The command should:
    1. Accept a question as $ARGUMENTS
    2. Load all persona YAML files from personas/ directory
    3. For each analyst persona: spawn an Agent with the persona's system_prompt, passing the question
       - Use subagent_type not specified (general-purpose) since these are reasoning agents
       - Spawn ALL analysts in parallel (one Agent tool call per analyst)
       - Each agent prompt: the system_prompt from YAML + "Question: {question}" + "Respond with structured output: position, confidence (0.0-1.0), evidence (list), challenges (list), questions (list)"
    4. Collect all analyst responses
    5. For each editor persona: spawn an Agent with the editor's system_prompt + all analyst output as context
       - Editors run sequentially (each sees prior editor output too)
       - Editor prompt: system_prompt + "Analyst perspectives: {all_analyst_output}" + "Respond with: blind_spots, synthesis, question_shifts, mechanisms"
    6. Format final output as markdown report:
       ```
       # Deliberation: {question}

       ## Analyst Perspectives
       ### {persona_name} ({reasoning_style})
       **Position:** ...
       **Confidence:** X.X
       **Evidence:** ...
       **Challenges:** ...
       **Questions:** ...

       ## Editorial Analysis
       ### {editor_name}
       **Blind Spots:** ...
       **Synthesis:** ...
       **Question Shifts:** ...

       ## Consensus & Dissent
       [Summary of where analysts agree/disagree]
       ```

    The slash command is a markdown file with instructions for Claude to follow.
    It should use the Agent tool for parallel execution.
    Include model routing: use model "opus" for ALL agents (analysts and editors). Deep reasoning with hard constraints requires the strongest model.
