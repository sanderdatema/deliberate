<objective>
## Goal
Add multi-round debate where analysts react to each other's output (Aslander's orkest-model), and structured fuzzy scoring output with confidence scores, evidence lists, and consensus/dissensie detection.

## Purpose
Phase 1 runs analysts independently — they never see each other. Aslander's core insight is that "stemmen reageren op elkaars output, niet alleen op de initiële vraag." Round 2 is where the real deliberation happens: analysts sharpen, challenge, and revise their positions based on what others said. Combined with fuzzy scoring (0.0-1.0 confidence), this enables automated detection of consensus vs. polarization.

## Output
- Updated `/deliberate` slash command with configurable multi-round support
- Updated persona YAML with structured output instructions
- Fuzzy scoring output format in analyst responses
- Consensus/dissensie detection in meta-analysis
- Updated tests
</objective>

## AC-1: Multi-Round Analyst Debate
```gherkin
Given the /deliberate command is invoked with a question
When analysts complete Round 1 (parallel, independent)
Then a Round 2 begins where each analyst receives a summary of ALL Round 1 output
And each analyst responds to the other perspectives while maintaining their hard constraints
And Round 2 analysts run in parallel
```

## AC-2: Structured Fuzzy Output
```gherkin
Given an analyst persona responds in any round
When their output is collected
Then it contains a clearly labeled confidence score (0.0-1.0)
And evidence items are listed individually
And challenges and questions are separated
And the output follows a consistent structure across all analysts
```

## AC-3: Consensus & Dissensie Detection
```gherkin
Given all analyst rounds and editor analyses are complete
When the meta-analysis is generated
Then it identifies points of convergence (where 3+ analysts agree)
And it identifies points of polarization (where analysts strongly disagree)
And it highlights the strongest question shifts from editors
And confidence scores are aggregated to show weighted consensus
```

## AC-4: Round Configuration
```gherkin
Given the /deliberate command
When invoked without flags
Then it runs 2 rounds by default (Round 1: independent, Round 2: reactive)
And the user can see which round is active during execution
```
