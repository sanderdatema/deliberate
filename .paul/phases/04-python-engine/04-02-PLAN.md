<objective>
## Goal
Build the async orchestration engine that runs a full deliberation using the Anthropic API directly. This is the core of the scripting engine — it replaces the slash command's orchestration logic with Python code that can be called programmatically.

## Purpose
The current orchestration lives in a prompt (`deliberate.md`). Moving it to Python enables: programmatic access, event streaming for web UI, proper testing with mocks, and CLI usage outside Claude Code.

## Output
- `deliberators/engine.py` with `DeliberationEngine` class
- Full deliberation flow: parallel analysts → compile → parallel R2 → sequential editors → samenvatter
- Event callback system emitting DeliberationEvent at each stage
- Fix from self-evaluation: Round 2 receives full R1 output (not lossy summary)
- Tests with mocked Anthropic API proving correct orchestration
</objective>

  <name>Task 1: Create the async orchestration engine</name>
  <files>deliberators/engine.py, deliberators/__init__.py</files>
  <action>
    **deliberators/engine.py:**

    Create `DeliberationEngine` class:

    ```
    class DeliberationEngine:
        def __init__(
            self,
            client: AsyncAnthropic,
            config: Config,
            personas: dict[str, Persona],
            on_event: Callable[[DeliberationEvent], Awaitable[None] | None] | None = None,
            on_text: Callable[[str, str], Awaitable[None] | None] | None = None,  # (agent_name, text_delta)
        )
    ```

    Core method:
    ```
    async def run(self, question: str, preset_name: str | None = None) -> DeliberationResult
    ```

    `DeliberationResult` dataclass (add to models.py or engine.py):
    - question: str
    - preset: Preset
    - rounds: dict[int, dict[str, str]]  — {round_num: {persona_name: output}}
    - editor_outputs: dict[str, str]  — {editor_name: output}
    - samenvatter_output: str | None

    Internal methods:
    - `_run_analyst_round(round_num, analysts, prior_round_output)` — parallel via asyncio.gather
    - `_run_editorial_round(editors, all_analyst_output)` — sequential, accumulative
    - `_call_agent(persona, prompt)` — single API call using client.messages.stream()
    - `_emit(event)` — calls on_event callback if set
    - `_build_analyst_prompt(persona, question, round_num, prior_output)` — constructs prompt
    - `_build_editor_prompt(persona, question, analyst_output, prior_editor_output)` — constructs prompt

    Prompt construction:
    - Round 1: system_prompt + question
    - Round 2: system_prompt + question + FULL Round 1 output (fixing the lossy summary bug)
    - Editors: system_prompt + question + all analyst output + prior editors' output
    - Samenvatter: system_prompt + question + everything

    API call pattern:
    ```python
    async with self.client.messages.stream(
        model="claude-sonnet-4-5-20250929",  # Use config.model to select
        max_tokens=4096,
        system=persona.system_prompt,
        messages=[{"role": "user", "content": prompt}],
    ) as stream:
        chunks = []
        async for text in stream.text_stream:
            chunks.append(text)
            if self.on_text:
                await maybe_await(self.on_text(persona.name, text))
        return "".join(chunks)
    ```

    Model mapping: config.model "opus" → "claude-opus-4-20250514", "sonnet" → "claude-sonnet-4-5-20250929"

    Event emission: emit DeliberationEvent at each stage transition (see AC-4).

    **deliberators/__init__.py:**
    - Add exports: DeliberationEngine, DeliberationResult

    Avoid:
    - Don't add CLI logic (that's plan 04-03)
    - Don't add HTTP/WebSocket server (that's Phase 5)
    - Don't compress Round 2 input — pass full output (this fixes the self-evaluation finding)
