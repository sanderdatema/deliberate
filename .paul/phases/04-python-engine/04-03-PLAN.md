<task type="auto">
  <name>Task 1: Create CLI entry point and formatter</name>
  <files>deliberators/__main__.py, deliberators/formatter.py</files>
  <action>
    **deliberators/__main__.py:**
    - Use argparse (no extra dependency)
    - Arguments:
      - `question` (positional) — the deliberation question
      - `--preset` (optional) — quick/balanced/deep, defaults to config default
      - `--config` (optional) — path to config.yaml, defaults to "config.yaml"
      - `--personas-dir` (optional) — path to personas/, defaults to "personas"
    - Load config and personas via ConfigLoader/PersonaLoader
    - Create AsyncAnthropic client (reads ANTHROPIC_API_KEY from env)
    - Create engine with on_event callback that prints progress to stderr
    - Run engine.run() via asyncio.run()
    - Format result via ResultFormatter and print to stdout
    - Exit 0 on success, 1 on error

    **deliberators/formatter.py:**
    - `ResultFormatter` class with `format(result: DeliberationResult) -> str`
    - Produces markdown matching the slash command's Step 7 output format:
      - `# Deliberatie: {question}`
      - `## Ronde N: ...` for each round
      - `### {analyst_name} — {reasoning_style}` for each analyst
      - `## Redactionele Analyse` with each editor
      - `## Kort & Concreet` with samenvatter output
    - Needs persona metadata (reasoning_style) — accept personas dict in constructor

    Avoid:
    - Don't add HTTP/WebSocket (Phase 5)
    - Don't add intake/clarification (that's the slash command's job)
    - Keep argparse simple — no subcommands, no fancy help formatting
  </action>
  <verify>uv run python -m deliberators --help 2>&1 | head -5</verify>
  <done>AC-1, AC-2 (CLI flags), AC-3 (formatter).</done>
</task>
