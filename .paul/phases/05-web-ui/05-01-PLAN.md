<objective>
## Goal
Build a web server with WebSocket streaming and a frontend that shows the deliberation in real-time — agent panels, round transitions, and streaming text as agents think.

## Purpose
The user wants to "see what the agents are thinking." The engine's event system (on_event + on_text) feeds directly into WebSocket messages. A single HTML page renders live agent panels.

## Output
- FastAPI server with WebSocket endpoint + REST endpoint to start deliberation
- Single-page HTML/JS frontend with real-time agent panels
- Streaming text visible as agents produce output
- Visual round progression and status indicators
</objective>

<task type="auto">
  <name>Task 1: Create FastAPI server with WebSocket streaming</name>
  <files>deliberators/web/__init__.py, deliberators/web/server.py, pyproject.toml</files>
  <action>
    **pyproject.toml:**
    - Add dependencies: fastapi, uvicorn[standard], websockets

    **deliberators/web/__init__.py:**
    - Empty init

    **deliberators/web/server.py:**
    - FastAPI app with:
      - `GET /` — serves index.html from static dir
      - `POST /api/deliberate` — accepts JSON `{question, preset}`, starts deliberation in background task, returns `{id}`
      - `WebSocket /ws/{deliberation_id}` — streams events as JSON to connected client
    - Use asyncio.Queue per deliberation to bridge engine events → WebSocket
    - Engine on_event callback pushes to queue
    - Engine on_text callback pushes `{type: "text_delta", agent_name, text}` to queue
    - Static file mount for CSS/JS
    - `__main__` block: `uvicorn.run(app, host="0.0.0.0", port=8000)`

    **deliberators/web/__main__.py:**
    - Entry point: `uv run python -m deliberators.web`
    - Starts uvicorn with the FastAPI app

    Flow:
    1. User opens browser → GET / → index.html
    2. User submits question → JS sends POST /api/deliberate → gets back {id}
    3. JS connects WebSocket /ws/{id}
    4. Server runs engine with callbacks pushing to queue
    5. WebSocket loop reads queue and sends JSON to browser
    6. Browser renders live panels

    Avoid:
    - No database, no sessions — one active deliberation at a time is fine for v0.2
    - No authentication
    - Don't over-engineer the WebSocket management — simple dict of {id: Queue}
  </action>

<task type="auto">
  <name>Task 2: Create frontend with live agent panels</name>
  <files>deliberators/web/static/index.html, deliberators/web/static/app.js, deliberators/web/static/style.css</files>
  <action>
    **index.html:**
    - Clean, minimal single-page app
    - Input form: question textarea + preset dropdown (quick/balanced/deep) + submit button
    - Status bar showing current phase (Ronde 1, Ronde 2, Redactie, etc.)
    - Agent panels container — panels appear dynamically as agents start
    - Final result area for Kort & Concreet + full report

    **app.js:**
    - On form submit: POST /api/deliberate, get ID, connect WebSocket /ws/{id}
    - Handle WebSocket messages by type:
      - `deliberation_started` → show status, clear panels
      - `round_started` → update status bar with round number
      - `agent_started` → create new panel for agent (with name header)
      - `text_delta` → append text to agent's panel (streaming effect)
      - `agent_completed` → mark panel as done (visual indicator)
      - `round_completed` → update status
      - `editorial_started` → update status to "Redactie"
      - `editorial_completed` → update status
      - `deliberation_completed` → show final result, enable new question
    - Auto-scroll panels as text streams in
    - Panels laid out in a grid (responsive)

    **style.css:**
    - Dark theme (comfortable for reading)
    - Agent panels: cards with header (agent name), scrollable content area, status indicator
    - Streaming text: monospace or readable font, subtle animation on new text
    - Status bar: fixed at top, shows current phase
    - Responsive grid: 2-3 columns on desktop, 1 on mobile
    - Compact but readable — the content matters more than the chrome

    Avoid:
    - No frameworks (React, Vue) — vanilla JS is sufficient
    - No build step — serve files directly
    - Don't over-style — functional first
  </action>
