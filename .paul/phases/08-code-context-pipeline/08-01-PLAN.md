<objective>
## Goal
Add a code context pipeline that reads codebase files and injects structured context into agent prompts. When a deliberation uses a code preset, agents receive the actual code they're reviewing.

## Purpose
Without code context, code personas can only discuss code in the abstract. This phase makes code review functional — agents see the actual files and can give specific, actionable feedback.

## Output
- `deliberators/context.py` — Code context builder (reads files, structures context)
- Updated `engine.py` — Accepts and injects code context into prompts
- Updated `models.py` — `code_context` field on DeliberationResult
- Updated `__main__.py` — `--files` CLI flag to pass code files
- Tests for context pipeline and prompt injection
</objective>

<task type="auto">
  <name>Task 1: Create CodeContextBuilder and update models</name>
  <files>deliberators/context.py, deliberators/models.py, tests/test_context.py</files>
  <action>
    **Create `deliberators/context.py`:**

    ```python
    class CodeContextBuilder:
        @staticmethod
        def build(paths: list[Path]) -> str | None
    ```

    - Accept a list of Path objects
    - For each path: read file content, detect language from extension
    - Format as:
      ```
      ## File: path/to/file.py (python)
      ```python
      [file contents]
      ```
      ```
    - Skip non-existent files (log warning)
    - Skip binary files (check for null bytes in first 1024 bytes)
    - Return None if no valid files found
    - Return the concatenated context string

    **Language detection:** Simple extension mapping (py→python, js→javascript, ts→typescript, yaml/yml→yaml, md→markdown, etc.). Default to plain text.

    **Update `deliberators/models.py`:**
    - Add `code_context: str | None = None` to DeliberationResult dataclass

    **Create `tests/test_context.py`:**
    - Test with valid Python file → correct formatting
    - Test with multiple files → all included
    - Test with non-existent file → skipped, returns remaining
    - Test with all non-existent → returns None
    - Test with binary file → skipped
    - Test language detection for common extensions
  </action>
  <verify>uv run pytest tests/test_context.py -v</verify>
  <done>AC-1 satisfied: CodeContextBuilder reads files and formats context</done>
</task>

<task type="auto">
  <name>Task 2: Inject code context into engine prompts and CLI</name>
  <files>deliberators/engine.py, deliberators/__main__.py, tests/test_engine.py</files>
  <action>
    **Update `deliberators/engine.py`:**

    1. Add `code_context: str | None = None` parameter to `run()` method
    2. Store on result: `result.code_context = code_context`
    3. Pass `code_context` to `_run_analyst_round()` and through to `_build_analyst_prompt()`
    4. Pass `code_context` to `_build_editor_prompt()`

    5. In `_build_analyst_prompt()`:
       - Add `code_context: str | None = None` parameter
       - After the QUESTION section, if code_context is not None:
         ```
         CODE UNDER REVIEW:
         {code_context}
         ```
       - This goes BEFORE round 2 perspectives (so analysts see code + other opinions)

    6. In `_build_editor_prompt()`:
       - Add `code_context: str | None = None` parameter
       - After ORIGINAL QUESTION, if code_context is not None:
         ```
         CODE UNDER REVIEW:
         {code_context}
         ```
       - This goes BEFORE analyst perspectives

    7. When code_context is None → no change to prompts (backward compatible)

    **Update `deliberators/__main__.py`:**
    - Add `--files` argument: `parser.add_argument("--files", nargs="+", type=Path, help="Code files to include as context")`
    - If `--files` provided: call `CodeContextBuilder.build(args.files)` and pass result to `engine.run(code_context=...)`
    - Import CodeContextBuilder from context module

    **Update `tests/test_engine.py`:**
    - Add test: engine with code_context → prompts contain "CODE UNDER REVIEW"
    - Add test: engine without code_context → prompts do NOT contain "CODE UNDER REVIEW"
    - Add test: code_context appears in both analyst and editor prompts
    - Existing tests must pass unchanged (they don't pass code_context, so None = no change)
  </action>
  <verify>uv run pytest -v — all tests pass including new prompt injection tests</verify>
  <done>AC-2, AC-3, AC-4, AC-5 satisfied: engine injects context, CLI accepts files, backward compatible</done>
