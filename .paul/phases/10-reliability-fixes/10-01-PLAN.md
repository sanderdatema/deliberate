<objective>
## Goal
Fix de 3 HIGH-priority bevindingen uit de code review: WebPusher connection churn, silent persona failures, en ontbrekende error handling in _call_agent.

## Purpose
Deze fixes verbeteren de betrouwbaarheid van de deliberatie-engine. WebPusher opent nu honderden kortstondige TCP-connecties per sessie. Ontbrekende personas worden stil genegeerd waardoor output incompleet is. Een enkele API-fout crasht de hele deliberatie.

## Output
- `deliberators/__main__.py`: WebPusher hergebruikt één httpx.AsyncClient
- `deliberators/loader.py`: Preset-validatie controleert of persona-keys bestaan
- `deliberators/engine.py`: _call_agent vangt API-fouten per agent op
- Tests voor alle drie de fixes
</objective>

<acceptance_criteria>

## AC-1: WebPusher hergebruikt HTTP client
```gherkin
Given een WebPusher instantie met actieve sessie
When meerdere push_event/push_text/push_result calls worden gedaan
Then wordt dezelfde httpx.AsyncClient hergebruikt (geen nieuwe client per call)
And de client wordt correct afgesloten na gebruik
```

## AC-2: Preset-validatie bij laden
```gherkin
Given een config.yaml met een preset die verwijst naar persona "nonexistent"
When ConfigLoader.load() of een nieuwe validatiefunctie wordt aangeroepen met de geladen personas
Then wordt een duidelijke fout gegooid die de ontbrekende persona benoemt
```

## AC-3: Persona-skip logging in engine
```gherkin
Given een engine run waar een persona-key niet in de personas dict voorkomt
When de analyst-ronde of editor-ronde draait
Then wordt een logging.warning() uitgestuurd met de ontbrekende persona-naam
And de deliberatie gaat door met de overige personas (geen crash)
```

## AC-4: Error handling per agent
```gherkin
Given een deliberatie met 5 analysts waar 1 API-call faalt (APIError, timeout, netwerk)
When _call_agent een exception gooit voor die ene agent
Then worden de resultaten van de andere 4 analysts behouden
And de gefaalde agent krijgt een foutmelding als output (niet een lege string)
And de deliberatie gaat door met editors
```

```gherkin
Given de nieuwe code
When uv run pytest wordt gedraaid
Then slagen alle bestaande tests (geen regressie)
And zijn er nieuwe tests voor AC-1 t/m AC-4
```
