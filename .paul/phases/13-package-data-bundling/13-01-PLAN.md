<objective>
## Goal
Move `personas/` and `config.yaml` inside the Python package as bundled data so the package is self-contained when installed via `uv tool install`.

## Purpose
First step toward global installability. Without bundled data, `uv tool install` produces a binary that can't find its personas or config. This phase makes the package self-contained.

## Output
- `deliberators/data/` directory containing `config.yaml` and `personas/` (copies of current root-level files)
- `pyproject.toml` updated with `[project.scripts]`, package-data inclusion, and version bump to 0.5.0
- Root-level `personas/` and `config.yaml` kept as-is (backward compatible for development)
- Helper function `get_data_path()` in `__init__.py` to locate bundled data
- All tests pass using the bundled data paths
</objective>

  <name>Task 2: Update pyproject.toml and __init__.py</name>
  <files>pyproject.toml, deliberators/__init__.py</files>
  <action>
    In pyproject.toml:
    1. Bump version to "0.5.0"
    2. Add `[project.scripts]` section:
       ```toml
       [project.scripts]
       deliberators = "deliberators.__main__:main"
       ```
    3. Add package-data inclusion (uv/hatch uses hatchling by default):
       ```toml
       [tool.hatch.build.targets.wheel]
       packages = ["deliberators"]
       ```
       Note: hatchling includes sub-packages and data files by default when they're inside the package directory. The `deliberators/data/` directory with its `__init__.py` will be included automatically.

    In deliberators/__init__.py:
    4. Add `get_data_path()` helper:
       ```python
       from pathlib import Path

       def get_data_path() -> Path:
           """Return the path to the bundled data directory."""
           return Path(__file__).parent / "data"
       ```
    5. Add `get_data_path` to `__all__`
  </action>
  <verify>uv run deliberators --help && uv run python -c "from deliberators import get_data_path; print(get_data_path())"</verify>
  <done>AC-2, AC-3, AC-4 satisfied: scripts entry point works, get_data_path() returns valid path, version is 0.5.0</done>
