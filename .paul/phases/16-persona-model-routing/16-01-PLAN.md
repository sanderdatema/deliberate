# Deliberators v0.6 — Adaptief Denksysteem

## Overview

Upgrade Deliberators van one-shot deliberatie naar een adaptief denksysteem met 5 werkstromen: persona-wijzigingen + model routing, intake fase, adaptive rounds, decision memory, en rapportage redesign. Dit plan dekt Phase 16: persona-wijzigingen en per-persona model routing — de fundering waar alle andere werkstromen op bouwen.

## Current State

- 25 persona YAML's in `personas/` — allemaal draaien op hetzelfde model (`config.yaml` → `model: opus`)
- Engine gebruikt `self.config.model` voor alle agents (`engine.py:163`)
- Persona dataclass heeft geen `model` veld (`models.py:10-19`)
- Schema valideert geen `model` veld (`personas/schema.yaml`)
- Config presets bevatten Lupin en Christensen (`config.yaml:11,39`)

### Key Discoveries

- `_subprocess_call()` gebruikt `self.config.model` hardcoded voor alle agents (`deliberators/engine.py:162`)
- Persona frozen dataclass heeft: name, role, reasoning_style, forbidden, focus, output_format, system_prompt (`deliberators/models.py:10-19`)
- PersonaLoader.load_all() leest YAML en maakt Persona objecten (`deliberators/loader.py`)
- Config.model is een string die naar subprocess gaat (`deliberators/models.py:40`)
- Presets refereren persona's by name string (`config.yaml:11`)

## What We're NOT Doing

- Intake fase (Phase 17) -- apart plan, afhankelijk van dit plan
- Adaptive rounds (Phase 18) -- apart plan, afhankelijk van Phase 17
- Decision memory (Phase 19) -- apart plan, onafhankelijk
- Rapportage redesign (Phase 20) -- apart plan, afhankelijk van 18+19
- Wijzigen van bestaande persona's (behalve Ive) -- alleen vervangen/toevoegen

## Implementation Approach

Verticale slice: alle wijzigingen voor per-persona model routing + persona swap in één fase. Van schema tot engine, inclusief tests. Volgorde: schema → models → loader → persona YAML's → config → engine → tests.

## Phase 1: Schema, Models & Loader

### Overview

Voeg `model` veld toe aan het persona-systeem: schema, dataclass, en loader.

### Changes Required:

#### 1. Persona Schema

**File**: `personas/schema.yaml`

Voeg optioneel `model` veld toe met enum `[opus, sonnet]`. Geen default in schema — persona YAML moet het expliciet specificeren.

#### 2. Persona Dataclass

**File**: `deliberators/models.py`

Voeg `model: str` toe aan de Persona frozen dataclass. Positie: na `name`, voor `role`.

```python
@dataclass(frozen=True)
class Persona:
    name: str
    model: str  # "opus" of "sonnet"
    role: Literal["analyst", "editor"]
    # ... rest unchanged
```

#### 3. PersonaLoader

**File**: `deliberators/loader.py`

Update `_load_persona()` om `model` veld uit YAML te lezen en door te geven aan Persona constructor. Valideer dat model in `("opus", "sonnet")`.

#### 4. Engine Model Routing

**File**: `deliberators/engine.py`

Wijzig `_subprocess_call()` om `persona.model` te gebruiken i.p.v. `self.config.model`:

```python
async def _subprocess_call(self, persona: Persona, prompt: str) -> str:
    model = persona.model or self.config.model  # fallback to global
    proc = await asyncio.create_subprocess_exec(
        "claude", "-p",
        "--model", model,
        ...
    )
```

Config.model blijft als fallback voor functionele agents (intake, convergentie) die geen persona hebben.

## Phase 2: Persona Swaps & Model Assignments

### Overview

Vervang Lupin door Machiavelli, Christensen door Knuth, herdefinieer Ive naar UI/UX, en voeg `model: opus|sonnet` toe aan alle 25 persona YAML's.

### Changes Required:

#### 1. Verwijder lupin.yaml, maak machiavelli.yaml

**File**: `personas/machiavelli.yaml`

Niccolo Machiavelli — Strategisch realist. Brengt het onaangename maar realistische perspectief. Niet contrair uit principe maar uit pragmatisch realisme. "Wat zou er gebeuren als mensen doen wat in hun eigenbelang is?"

- role: analyst
- model: opus
- reasoning_style: Strategisch realist — analyseert machtsdynamiek en eigenbelang als drijfveren
- forbidden: Idealisme zonder bewijs, moraliseren, aannemen dat mensen tegen hun eigenbelang handelen, negeren van machtsverhoudingen
- focus: Wat zou er gebeuren als iedereen puur uit eigenbelang handelt? Welke incentives sturen gedrag?

#### 2. Verwijder christensen.yaml, maak knuth.yaml

**File**: `personas/knuth.yaml`

Donald Knuth — Performance-analist. Beoordeelt algoritmische complexiteit, data structure keuzes, premature optimization vs. echte bottlenecks.

- role: analyst
- model: sonnet
- reasoning_style: Algoritmische analist — beoordeelt computationele complexiteit en performance trade-offs
- forbidden: Premature optimization zonder profiling data, negeren van Big-O, accepteren van brute-force als er betere structuren zijn, performance-claims zonder benchmarks
- focus: Algoritmische complexiteit, data structures, hot paths, memory footprint

#### 3. Herdefinieer jony-ive.yaml naar UI/UX

**File**: `personas/jony-ive.yaml`

Jony Ive — UI/UX design reviewer. Beoordeelt de **gebruikerservaring** die code produceert: visuele hierarchie, interactiepatronen, toegankelijkheid, design system naleving, responsive gedrag.

- reasoning_style: UI/UX design reviewer — beoordeelt de gebruikerservaring die code produceert
- forbidden: Code-formatting/whitespace beoordelen (dat is Linus' domein), negeren van accessibility, accepteren van inconsistente UI patronen, technische implementatie boven gebruikerservaring stellen
- focus: Wat ervaart de eindgebruiker? Visuele consistentie, interactieflow, toegankelijkheid, responsiveness

#### 4. Model assignments voor alle persona's

Alle 25 persona YAML's krijgen een `model:` veld. Toewijzingen per audit:

**Opus** (genuanceerd redeneren, synthese, creatief):
- socrates, da-vinci, machiavelli, templar, weil, noether, ibn-khaldun

**Sonnet** (gestructureerd, checklist, pattern matching):

#### 5. Update config.yaml presets

**File**: `config.yaml`

- quick: occam, holmes, **machiavelli** (was: lupin)
- balanced: socrates, occam, da-vinci, holmes, **machiavelli** (was: lupin)
- deep: socrates, occam, da-vinci, holmes, **machiavelli**, templar, tubman, ibn-khaldun (was: lupin)

- code_deep: linus, kent-beck, fowler, schneier, jobs, don-norman, **jony-ive**, **knuth**, hopper (was: christensen, ive had andere rol)

- `test_personas.py`: schema validatie voor alle persona's inclusief nieuw `model` veld

### Integration Tests

- Engine.run() met mixed Opus/Sonnet persona's (via mock agent_fn die model checkt)
- Preset validatie met nieuwe samenstelling

### Manual Testing Steps

1. `uv run pytest -v` — alle tests groen
