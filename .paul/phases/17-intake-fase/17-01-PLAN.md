<objective>
## Goal

Voeg een functionele intake-agent toe die vóór de analyst-rondes de vraag analyseert op helderheid, impliciete aannames en missende context. De agent produceert een intake-brief die meegaat naar alle analysts als contextuele instelling.

## Purpose

Analysts krijgen nu een vraag zonder context: ze moeten zelf aannames maken over wat de gebruiker bedoelt. De intake-brief zorgt dat alle analysts dezelfde lens hebben: wat is de eigenlijke vraag, welke dimensies zijn relevant, welke aannames zijn in het spel. Dit verbetert de kwaliteit en coherentie van de deliberatie significant.

## Output

- `IntakeBrief` dataclass in `models.py`
- `_run_intake()` methode in `DeliberationEngine`
- `on_clarify` callback type voor interactieve verduidelijking (max 3 rondes)
- `_call_functional_agent()` voor subprocess calls zonder Persona
- `engine.run()` roept intake aan vóór analyst-rondes
- `_build_analyst_prompt()` injecteert intake-brief als context
- CLI: intake events naar stderr, `on_clarify` via stdin
- `intake_started` / `intake_completed` events in DeliberationEvent

</objective>


    6. Add `_run_intake(question: str) -> IntakeBrief` async method:

       - Build initial prompt: f"QUESTION:\n{question}"

         c. If is_clear == "yes" OR no on_clarify OR iteration >= 3: break
         d. If is_clear == "no" AND on_clarify AND clarification_question != "none":

            - Append to clarifications list: (clarification_question, user_answer)
            - Rebuild prompt: original question + "\n\nADDITIONAL CONTEXT:\n" + joined Q&A pairs
       - Build IntakeBrief(question=question, summary=parsed["brief"], clarifications=tuple(clarifications), is_clear=(parsed["is_clear"]=="yes"))
       - Emit intake_completed with data={"is_clear": brief.is_clear, "had_clarifications": len(clarifications) > 0}
       - Return brief
