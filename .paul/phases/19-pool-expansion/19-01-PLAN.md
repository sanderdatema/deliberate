<objective>
## Goal
Add `domains` field to the persona system (schema, dataclass, loader) and tag all 25 existing personas with domain expertise tags. This is the foundation for Phase 20's dynamic team selection — the intake agent needs domain tags to match personas to questions.

## Purpose
Phase 20's dynamic team selection requires machine-readable expertise metadata on each persona. Without `domains`, the intake agent has no structured way to select the best team for a given question. This plan adds the field everywhere and tags existing personas, so Plan 02 (new persona creation) and Phase 20 can build on it.

## Output
- `domains` field in schema.yaml (required, list of strings, min 1)
- `domains: tuple[str, ...]` in Persona dataclass
- PersonaLoader reads and validates domains
- All 25 existing persona YAMLs tagged with domains
- Updated tests for the new field
</objective>
