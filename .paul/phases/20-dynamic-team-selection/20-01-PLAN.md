# Phase 20 Plan 01: Dynamic Team Selection

**Intake agent wordt teamsamensteller — één unified /deliberate command, presets als pool-hints, cross-domein teams met genderbalans**

## Context

Phase 19 delivered 54 domain-tagged personas. Currently, presets define fixed analyst/editor lists — the intake agent cannot compose teams dynamically. Two separate commands (/deliberate and /deliberate-code) exist with duplicated logic. This phase makes the intake agent a team assembler and merges everything into one command.

## Acceptance Criteria

| ID | Criterion | Verify |
|----|-----------|--------|
| AC-1 | TeamSelectionAgent selects analysts + editors from full pool based on question + domains | Unit test: mock subprocess returns valid team, engine uses selected personas |
| AC-2 | Gender balance enforced: selected team has ≥40% of each binary gender when pool allows | Unit test: selection prompt includes gender balance instruction |
| AC-3 | Preset model uses team_size + editor_count instead of fixed analysts/editors tuples | Config validation test: presets have team_size, analysts/editors optional (backward compat) |
| AC-4 | code_* presets removed; --files flag works with any preset | CLI test: --files accepted without code_ prefix, old code_* presets gone |
| AC-5 | /deliberate-code removed; /deliberate handles both general and code review | Slash command file test: only deliberate.md exists, supports --files |
| AC-6 | Intake runs for all questions (no more code_* skip) | Engine test: intake not skipped regardless of preset |
| AC-7 | All existing tests pass (updated counts/assertions) | pytest: 0 failures |

## Tasks

### Task 1: Team Selection Agent + Model Changes

**Action:**

1. **models.py** — Update Preset dataclass:
   - Add `team_size: int` field (number of analysts to select)
   - Add `editor_count: int` field (number of editors to select)
   - Keep `analysts: tuple` and `editors: tuple` as optional override (if set, skip team selection — backward compat during transition)
   - Add `domains_hint: tuple[str, ...] = ()` field (optional domain bias for selection)

2. **config.yaml** — Redefine presets:
   ```yaml
   presets:
     quick:
       team_size: 3
       editor_count: 1
       rounds: 1
     balanced:
       team_size: 5
       editor_count: 2
       rounds: 2
     deep:
       team_size: 8
       editor_count: 3
       rounds: 2
   ```
   Remove `code_quick`, `code_balanced`, `code_deep` presets entirely. Remove fixed `analysts` and `editors` lists from all presets.

3. **loader.py** — Update ConfigLoader:
   - Remove `validate_preset_personas()` (no longer validating fixed persona lists)
   - Add validation for team_size > 0, editor_count > 0
   - Handle backward compat: if preset has `analysts`/`editors` keys, load them; otherwise None

4. **engine.py** — Add `_run_team_selection()`:
   - New functional agent call via `_call_functional_agent()`
   - System prompt receives:
     - Full persona catalog: `name | domains | gender | role` for all 54 personas
     - `team_size` and `editor_count` from preset
     - The user's question + intake brief (if available)
     - `domains_hint` from preset (if any)
     - Gender balance instruction: "Select a team with at least 40% representation of each binary gender (M/F). Include non-binary personas where relevant."
   - Expected output format: `ANALYSTS: name1, name2, ...\nEDITORS: name1, name2, ...\nREASON: ...`
   - Parse output, resolve names to Persona objects via PersonaLoader
   - Fallback: if parsing fails or names don't resolve, fall back to preset's fixed lists (if any) or raise error
   - Add `team_selected` event to DeliberationEvent enum

5. **engine.py** — Update `run()` method:
   - After intake, call `_run_team_selection()` to get analysts + editors
   - Use selected team instead of preset's fixed lists
   - If preset has fixed `analysts`/`editors` (override), skip team selection
   - Remove the `if preset_name.startswith("code_"):` intake skip logic

6. **models.py** — Add `team_selected` to DeliberationEvent enum

7. **tests/** — Update tests:
   - `test_engine.py`: Add test for `_run_team_selection()` mock, verify team_selected event, verify intake always runs
   - `test_config.py`: Update preset validation tests (team_size/editor_count instead of fixed lists), remove code_* preset tests
   - `test_loader.py`: Update loader tests for new preset format, remove validate_preset_personas tests
   - `test_quality.py`: Update any preset-dependent quality tests

**Verify:**
```bash
uv run pytest tests/test_engine.py tests/test_config.py tests/test_loader.py -v
```

**Done when:** TeamSelectionAgent selects team from pool, presets use team_size/editor_count, code_* presets removed, all tests pass.

## Risk Assessment

| Risk | Impact | Mitigation |
|------|--------|------------|
| Team selection LLM output parsing fails | Teams can't be assembled | Robust parsing with fallback; clear output format instructions |
| Removing code_* presets breaks existing workflows | Users get errors | --files flag preserved, team selector handles code context |
| Gender balance constraint limits team quality | Suboptimal team for narrow domains | 40% threshold (not 50%), "where pool allows" escape clause |
